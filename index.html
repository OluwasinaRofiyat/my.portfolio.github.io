Clickhouse Tripdata Project

NYC Trip Data ETL Pipeline

This project demonstrates an end-to-end ETL pipeline developed to process and aggregate NYC trip data for data warehousing and analytical reporting. The pipeline utilizes a ClickHouse database as the data source and PostgreSQL for data storage and transformation, creating a seamless data flow from extraction to staging and production tables.

Project Overview:

  • Data Extraction: Using ClickHouse SQL, the pipeline fetches specific trip data (pickup date, vendor ID, passenger count, trip distance, payment type, fare amount, and tip amount) based on given conditions. This data is then saved as a CSV file for further processing.
  • Data Loading: The extracted data is loaded into the staging area of a PostgreSQL database, utilizing SQLAlchemy and Pandas to handle CSV imports efficiently.
  • Data Transformation: A stored procedure, agg_tripdata(), aggregates daily trip data and performs calculations such as total and average passenger count, trip distance, fare amount, and tips. It logs the pipeline execution status for monitoring purposes.
  • Production Tables: The stored procedure populates production tables daily_agg_tripdata and daily_agg_tripdata_channel, enabling analytical queries on trip metrics by day and payment type.

Key Technologies:

  • Python, SQLAlchemy, Pandas
  • ClickHouse, PostgreSQL, Stored Procedures (PL/pgSQL)
  • SQL, Environment Variables for Secure Configurations

This ETL pipeline project highlights my expertise in data extraction, transformation, and loading processes, as well as experience in working with multiple databases and implementing data aggregation methods to support efficient analytical queries. The result is a structured, scalable pipeline ready for integration into broader data analytics or business intelligence applications.

NYC PAYROLL PIPELINE

NYC Payroll Data Integration Pipeline

This project implements a comprehensive data pipeline to analyze payroll data for the City of New York, focusing on financial resource allocation and public transparency.

Project Objectives

  • Automate the data ingestion and transformation of payroll records
  • Optimize data storage for efficient querying and analysis
  • Enable transparent reporting on public payroll spending

Pipeline Architecture

The pipeline integrates various AWS services including AWS Glue, AWS Step Functions, and AWS S3, creating an end-to-end solution that ingests, processes, and loads data into a cloud data warehouse for analysis.

Steps in the Pipeline

  1. Data Preprocessing: Concatenated payroll_2020 and payroll_2021 files in VSCode, then used AWS Crawlers for data cataloging.
  2. Data Transformation: AWS Glue jobs to normalize and transform data into analytics-ready tables.
  3. Data Aggregation: SQL scripts to create aggregation tables for financial and trend analysis.
  4. Public Accessibility: Built views for accessible payroll summaries, supporting open data initiatives.

Key Technologies

Data Storage: AWS S3

Data Processing: AWS Glue, SQL

Orchestration: AWS Step Functions

Outcomes

Resulted in a scalable, automated pipeline capable of handling future payroll data, enabling valuable insights for both financial analysis and public transparency.
